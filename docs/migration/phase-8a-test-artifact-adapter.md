# Phase 8A test artifact adapter

## Purpose

Phase 8A adds a test-only artifact readiness layer after the Phase 8 dry-run writer adapters.

It answers this question:

```text
Can the shadow export models safely produce artifact bytes, text, or files through isolated writer calls?
```

The default Phase 8A answer is deterministic and conservative: RVM, ATT, and GLB artifacts are not generated unless a safe bridge is explicitly implemented and proven.

## Pipeline

```text
RvmExportModel.v1
+ AttExportModel.v1
+ GlbVisualModel.v1
+ ExportModelCompilationAudit.v1 ok
+ WriterAdapterPlan.v1
+ WriterAdapterAudit.v1 ok
-> TestArtifactAdapterPlan.v1
-> TestArtifactAdapterAudit.v1
```

## Why Phase 9 is premature

Phase 9 canvas or diagnostic preview should not consume uncertain artifact state. Phase 8A creates deterministic artifact-readiness status first, including explicit blocked reasons for artifact paths that are not safe yet.

## Test-only boundary

Phase 8A is not runtime wiring, app wiring, canvas preview, browser export, production converter replacement, or user-visible download.

It does not modify production writer files, existing runtime files, browser behavior, Pages cache keys, or current RVM/ATT/GLB output.

The test-only adapter lives under `src/artifact-adapters/` and is invoked only by tests or future explicit test-only scripts.

## Side-effect policy

Default Phase 8A has no:

- browser download;
- object URL;
- DOM usage;
- browser global usage;
- preview surface usage;
- runtime scene object creation;
- file output;
- production path mutation;
- cache-key mutation.

Generated artifact payloads are not committed to the repo. If a future test-only bridge emits bytes or text, it must report compact metadata such as length and checksum rather than large inline payload dumps.

## RVM readiness

Phase 7 and Phase 8 still report:

```text
transformApplied: false
```

Therefore Phase 8A must not produce an RVM artifact and mark it ready. The RVM artifact remains blocked by default with this reason:

```text
RVM artifact blocked until final review transform policy is implemented
```

Logical writer chunks may still be traced, but artifact readiness remains false.

## ATT bridge result

No safe bridge from `AttExportModel.v1` / `WriterAdapterPlan.v1` into the production ATT writer model is implemented in Phase 8A.

The default ATT artifact status is blocked with this reason:

```text
ATT writer adapter requires production writer model bridge not implemented in Phase 8A
```

## GLB bridge result

No clean writer-independent GLB artifact path exists in Phase 8A.

The default GLB artifact status is blocked with this reason:

```text
GLB test artifact writer not implemented in Phase 8A
```

No runtime geometry, GLTF JSON, or GLB bytes are created.

## Source trace rule

Phase 8A creates deterministic source trace entries for planned, blocked, and deferred items.

For BM_CII, trace coverage is expected to be:

```text
sourceTraceCount === 52
tracedStraightPipeCount === 19
tracedBlockedFlangeCount === 8
tracedBlockedValveCount === 6
tracedBlockedBendCount === 7
tracedDeferredSupportCount === 12
```

The trace does not require generated bytes to be valid.

## Blocked and deferred preservation

Blocked flanges, valves, and bends remain blocked. Phase 8A does not fabricate RVM records, TORUS/code4 bends, flange or valve placeholder cylinders, support primitives, fallback boxes, fallback spheres, fallback pyramids, or bend-center geometry from chord midpoint evidence.

Supports remain deferred.

## Handoff options

After Phase 8A, the project can proceed to one of these choices:

- Phase 9 diagnostic/canvas preview adapter, if artifact state is deterministic enough for UI presentation;
- Phase 8B final review transform policy, to unblock RVM artifact readiness;
- Phase 8C isolated writer bridge for a real artifact subset, with explicit test-only controls.
